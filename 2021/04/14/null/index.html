[JS] localStorage 使用方法

localstorage 是存儲

在了解 cookie 與 webStorage 前，首先要知道，**HTTP 是一個無狀態協議 ( Stateless Protocol )**，意旨客戶端 ( Client ) 對伺服器 ( Sever ) 所發出的請求 ( request ) 是獨立的，在每次的請求中是無法得知上一次請求的內容與資訊。

cookie

因為 HTTP 的無狀態協議，當之後需要處理前面的資訊，必需再附上之前的資訊，造成需要傳送的資料量增大，因此在 1993 年，Lou Montulli 發明 Cookie，而 Cookie 是由伺服器發給客戶端一小型文字檔瀏覽器中，而瀏覽器在每一次發送請求時，都會在 Header 中設定 Cookie 屬性，讓伺服器能經由 Cookie 的訊息得知使用者資訊。

一、關於 Cookie 與 Web Storage

在 HTML5 之前，是使用 Cookie 來儲存用戶資料，在 HTML5，加入 Web Storage 概念，Web Storage 可將用戶資料儲存在本地端，而 Cookie 與 Web Storage 差異如下：

  1. 建值對
  2. Cookie 儲存空間約 4k，而 Web Storage 儲存資料比 Cookie 還大，一般有 5mb 以上
  3. Cookie

2. Web Storage 包含兩種機制

  • sessionStorage

  • localStorage
    沒有過期時間 (關掉瀏覽器分頁後，資料依然存在)，除非主動刪除
    儲存在客戶端，適合處理不敏感的資料
